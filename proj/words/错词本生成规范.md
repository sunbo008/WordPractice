# 错词本生成规范

## 一、文件命名规则

文件名格式：`YYYY-MM-DD-HHMMSS.json`

**示例**：
- `2025-10-22-221816.json`
- `2025-11-20-153045.json`

**说明**：
- `YYYY`：四位年份
- `MM`：两位月份（01-12）
- `DD`：两位日期（01-31）
- `HHMMSS`：时分秒，无间隔符

---

## 二、文件保存路径

```
proj\words\MissedWords\
```

---

## 三、JSON 格式结构

```json
{
  "words": [
    {
      "word": "see",
      "phonetic": "[siː]",
      "meaning": "看见",
      "difficulty": 1
    },
    {
      "word": "tree",
      "phonetic": "[triː]",
      "meaning": "树",
      "difficulty": 1
    }
  ]
}
```

---

## 四、字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `word` | string | 是 | 英文单词（小写） |
| `phonetic` | string | 是 | 音标（使用方括号包裹） |
| `meaning` | string | 是 | 中文释义（主要含义） |
| `difficulty` | number | 是 | 难度等级（1-5） |

**难度等级参考**：
- 1：基础词汇（小学水平）
- 2：常用词汇（初中水平）
- 3：中等词汇（高中水平）
- 4：进阶词汇（大学水平）
- 5：高级词汇（专业/学术）

---

## 五、使用说明

### 1. 提供生词格式

向 AI 助手提供生词时，可以使用以下任意格式：

**格式一：简单列表**
```
see - 看见
tree - 树
apple - 苹果
```

**格式二：带音标**
```
see [siː] - 看见
tree [triː] - 树
apple [ˈæpl] - 苹果
```

**格式三：完整信息**
```
单词：see
音标：[siː]
释义：看见
难度：1
```

### 2. 生成命令示例

```
将以下生词生成错词本：
see - 看见
tree - 树
apple - 苹果
```

AI 助手会自动：
1. 补充音标（如未提供）
2. 评估难度等级（如未提供）
3. 生成符合规范的 JSON 文件
4. 保存到 `proj\words\MissedWords\` 目录

---

## 六、注意事项

1. **文件名唯一性**：每次生成使用当前时间戳，确保不重复
2. **单词格式**：英文单词统一使用小写
3. **音标格式**：使用国际音标（IPA），用方括号 `[]` 包裹
4. **中文释义**：提供最常用的 1-2 个主要含义，用顿号或斜杠分隔
5. **字符编码**：文件使用 UTF-8 编码
6. **JSON 格式**：必须是有效的 JSON 格式，注意最后一项无逗号
